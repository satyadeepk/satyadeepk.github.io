I and some of my friends encountered the following error when I tried ESC/JAVA 2 on open-jdk 1.6 in Ubuntu. Using Sun JDK 1.6 also caused the same error.

[bash]/usr/java/jre1.6.0_18/lib/rt.jar:java/io/PrintStream.class: Error: Different or incompatible return types on overridden(hidden) and overriding(hiding) methods[/bash]

One solution which worked for me is install Sun JDK 1.5  as given below and use it for ESC/JAVA. This is because ESC/JAVA 2 uses JML specification of JDK 1.4. Hence it may cause trouble in case of higher versions of JDK. (See Issues of ESC/JAVA 2 with JDK 1.6 ).

Steps to install Sun JDK 1.5 in Linux:

Note that any previous installation of JDK 1.6 may be left as it is.

1) Download JDK 1.5 for Linux from here or Direct Link .

2) Change to the directory where you downloaded the file and change the permission of the executable.

[bash]chmod +x jdk-1_5_0_22-linux-i586.bin [/bash]

3) Run the self extracting binary file

[bash] ./jdk-1_5_0_22-linux-i586.bin [/bash]

This will create a directory named  jdk1.5.0_22 in the current working directory.

4) Move this directory to /usr/java/ by

[bash] sudo mv jdk1.5.0_22 /usr/java/jdk1.5.0_22  [/bash]

5) Set up the environmental variables JAVA_HOME and PATH.

Open files  ~/.bashrc or ~/.bash_profile files present in your home directory and append the following lines at the end. This makes the bash terminal to use JDK 1.5

[bash]
JAVA_HOME="/usr/java/jdk1.5.0_22"
export JAVA_HOME
export PATH=$JAVA_HOME/bin:$PATH
[/bash]

6) Close the terminal and open it again and run the following command to check the java version

[bash]javac -version
java -version[/bash]
